### 17 - Нарушен принцип "представление-функционал"
**Описание:** Нарушен принцип "представление-функционал", согласно которому обращения к DOM должны осуществляться по id/data-attr, классы же служат только для стилизации. https://drive.google.com/uc?id=1EZGrU20JJec7pguzgf2zdcBjMfYQJ4fq

**Исправлено:** [ ]

### 18 - Отдельные методы для обработчиков событий

**Исправлено:** [ ]

### 22 - Неккоректный выбор валидатора
**Описание:** Выбрана верная логика валидации, но она некорректно исполнена по части выбора валидатора. Не следует обращаться по классу поля, будет уместнее использование data-attr. Например, data-input-required="type", где "type" будет означать способ валидации данного поля. https://drive.google.com/uc?id=1xzoBtOIUeJh7RKroX3vjHe5UWvhxy0KQ

**Исправлено:** [ ]

### 23 - Нарушение принципов SOLID
**Описание:** Класс "ValidateForm" дословно означает "валидация формы". Почему тогда в нем присутствует логика по отправке на backend? Это нарушает принципы SOLID. Следует выделить еще один модуль, который отправляет данные.

**Исправлено:** [ ]
